# 📌 3.4 — Operators
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## ① DEFINITION

**Plain English:** Operators are symbols that tell JavaScript to perform a specific operation on values — arithmetic, comparison, logical operations, and more. JavaScript has a rich set of operators, including some unique ones like nullish coalescing (`??`), optional chaining (`?.`), and the spread/rest operator (`...`).

**Technical Definition:** Operators are built-in functions defined by the ECMAScript specification that accept operands and produce a result. They have defined **precedence** (which runs first) and **associativity** (left-to-right or right-to-left for equal precedence).

**One-line mental model:**
> *Operators transform values. Learn `??`, `?.`, `...`, and destructuring — they're the most important modern operators.*

## ② DRILL DOWN

### 🔹 A. Arithmetic Operators

```javascript
// Basic:
5 + 3;   // 8 (addition)
5 - 3;   // 2 (subtraction)
5 * 3;   // 15 (multiplication)
5 / 3;   // 1.6666... (division — always float!)
5 % 3;   // 2 (remainder/modulo)
5 ** 3;  // 125 (exponentiation, ES2016)

// Increment/Decrement:
let x = 5;
x++;     // Returns 5, then x becomes 6 (post-increment)
++x;     // x becomes 7, then returns 7 (pre-increment)
x--;     // Returns 7, then x becomes 6
--x;     // x becomes 5, then returns 5

// Unary operators:
+true;   // 1 (coerces to number)
+"42";   // 42
-5;      // -5 (negation)
-"-5";   // 5 (negation + coercion)

// Assignment operators:
x += 5;  // x = x + 5
x -= 3;  // x = x - 3
x *= 2;  // x = x * 2
x /= 4;  // x = x / 4
x %= 3;  // x = x % 3
x **= 2; // x = x ** 2
x &&= 5; // x = x && 5 (ES2021 — assign only if x is truthy)
x ||= 5; // x = x || 5 (assign only if x is falsy)
x ??= 5; // x = x ?? 5 (assign only if x is null/undefined)
```

### 🔹 B. Logical Operators

```javascript
// AND (&&) — returns first falsy value or last truthy value:
true && "hello";    // "hello" (both truthy → return last)
false && "hello";   // false (first is falsy → return it)
"" && "hello";      // "" (empty string is falsy)
"hi" && "there";    // "there" (both truthy → last value)
0 && "never";       // 0

// OR (||) — returns first truthy value or last falsy value:
false || "hello";   // "hello"
"" || "default";    // "default"
0 || 42;            // 42
"exists" || "default"; // "exists"
null || undefined || "final"; // "final"

// NOT (!) — returns the inverse boolean:
!true;    // false
!0;       // true
!"";      // true
!!"hello"; // true (double negation → boolean conversion)

// Short-circuit evaluation:
false && expensiveFunction(); // expensiveFunction is NEVER called
true || expensiveFunction();  // expensiveFunction is NEVER called

// Practical pattern — guard clauses:
const user = null;
const name = user && user.name; // null (short-circuits — doesn't access .name)
```

### 🔹 C. Bitwise Operators

```javascript
// These operate on 32-bit integers (values are converted to 32-bit signed integers):
5 & 3;   // 1   (AND: 101 & 011 = 001)
5 | 3;   // 7   (OR:  101 | 011 = 111)
5 ^ 3;   // 6   (XOR: 101 ^ 011 = 110)
~5;      // -6  (NOT: inverts all bits, including sign)
5 << 1;  // 10  (Left shift: 101 → 1010)
5 >> 1;  // 2   (Right shift: 101 → 10)
-5 >>> 1; // 2147483645 (Unsigned right shift — fills with 0, not sign bit)

// Practical uses:
// Fast integer conversion (truncates to 32-bit integer):
~~3.7;       // 3 (double NOT — faster than Math.floor for positive numbers)
3.7 | 0;     // 3 (OR with 0 — same effect)
3.7 >> 0;    // 3 (right shift by 0)

// Check if number is even/odd:
7 & 1;  // 1 (odd)
8 & 1;  // 0 (even)

// Toggle a flag:
let flags = 0b1010;
flags ^= 0b0010; // Toggle bit 1: 0b1000

// Color manipulation (RGB):
const color = 0xFF5500;
const r = (color >> 16) & 0xFF; // 255
const g = (color >> 8) & 0xFF;  // 85
const b = color & 0xFF;          // 0
```

### 🔹 D. Nullish Coalescing (`??`) — ES2020

```javascript
// ?? returns the right-hand side ONLY when the left is null or undefined
// Unlike ||, it does NOT treat 0, "", or false as "empty"

const value = 0;
value || "default"; // "default" (0 is falsy → || triggers)
value ?? "default"; // 0 (0 is NOT null/undefined → ?? keeps it)

const text = "";
text || "fallback"; // "fallback" (empty string is falsy)
text ?? "fallback"; // "" (empty string is NOT null/undefined)

// Perfect for configuration with valid falsy values:
const config = {
    port: 0,        // Valid! 0 means "auto-assign"
    debug: false,    // Valid! Explicitly disabled
    name: "",        // Valid! Empty string
    timeout: null,   // Missing! Should use default
};

const port = config.port ?? 3000;       // 0 (keeps the valid 0)
const debug = config.debug ?? true;     // false (keeps the explicit false)
const name = config.name ?? "default";  // "" (keeps the empty string)
const timeout = config.timeout ?? 5000; // 5000 (null → use default) ✅

// Cannot mix ?? with && or || without parentheses:
// a || b ?? c;    // SyntaxError!
(a || b) ?? c;    // OK
a || (b ?? c);    // OK
```

### 🔹 E. Optional Chaining (`?.`) — ES2020

```javascript
// ?. safely accesses nested properties — returns undefined if any part is null/undefined

const user = {
    address: {
        city: "NYC"
    }
};

// Without optional chaining:
const city = user && user.address && user.address.city; // "NYC"

// With optional chaining:
const city2 = user?.address?.city; // "NYC"

// If any part is missing:
const zip = user?.address?.zip;      // undefined (no error!)
const phone = user?.phone?.mobile;   // undefined (phone doesn't exist)

// Optional method call:
const arr = [1, 2, 3];
arr.find?.(x => x > 2);       // 3 ✅
arr.findWrong?.(x => x > 2);  // undefined (method doesn't exist, no error)

// Optional bracket access:
const map = { a: 1 };
const key = "b";
map?.[key]; // undefined

// Short-circuits:
null?.x.y.z; // undefined (does NOT throw — stops at null)
undefined?.method(); // undefined (does NOT call method)

// Combined with ??:
const displayName = user?.profile?.name ?? "Anonymous";
```

### 🔹 F. Destructuring

```javascript
// Object destructuring:
const { name, age } = { name: "Alice", age: 30, city: "NYC" };
// name = "Alice", age = 30

// With renaming:
const { name: userName, age: userAge } = user;

// With defaults:
const { name = "Anonymous", role = "user" } = {};
// name = "Anonymous", role = "user"

// Nested:
const { address: { city, zip = "00000" } } = user;

// Array destructuring:
const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first = 1, second = 2, rest = [3, 4, 5]

// Skipping elements:
const [, second2] = [1, 2, 3];
// second2 = 2

// Swap variables:
let a = 1, b = 2;
[a, b] = [b, a]; // a = 2, b = 1

// Function parameter destructuring:
function greet({ name, greeting = "Hello" }) {
    console.log(`${greeting}, ${name}!`);
}
greet({ name: "Alice" }); // "Hello, Alice!"

// Destructuring in loops:
const users = [{ name: "A", age: 20 }, { name: "B", age: 30 }];
for (const { name, age } of users) {
    console.log(name, age);
}
```

### 🔹 G. Spread/Rest Operator (`...`)

```javascript
// SPREAD — expands an iterable into individual elements:

// Array spread:
const arr1 = [1, 2, 3];
const arr2 = [0, ...arr1, 4]; // [0, 1, 2, 3, 4]

// Object spread (shallow copy):
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

// Override properties:
const defaults = { color: "red", size: "medium" };
const custom = { ...defaults, color: "blue" }; // { color: "blue", size: "medium" }

// Function call spread:
Math.max(...[1, 2, 3]); // 3 (spreads array as arguments)

// REST — collects remaining elements:

// In functions:
function sum(...numbers) {
    return numbers.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4); // 10

// In destructuring:
const { a, ...remaining } = { a: 1, b: 2, c: 3 };
// a = 1, remaining = { b: 2, c: 3 }

const [head, ...tail] = [1, 2, 3, 4];
// head = 1, tail = [2, 3, 4]
```

### 🔹 H. Other Important Operators

```javascript
// delete — removes a property from an object:
const obj = { a: 1, b: 2 };
delete obj.a; // true (removes property, may switch to dictionary mode in V8)
console.log(obj); // { b: 2 }

// typeof — returns type as string:
typeof 42;          // "number"
typeof "hello";     // "string"
typeof true;        // "boolean"
typeof undefined;   // "undefined"
typeof null;        // "object" (bug!)
typeof {};          // "object"
typeof [];          // "object"
typeof function(){}; // "function"
typeof Symbol();    // "symbol"
typeof 42n;         // "bigint"

// instanceof — checks prototype chain:
[] instanceof Array;   // true
[] instanceof Object;  // true (Array inherits from Object)
"hi" instanceof String; // false (primitive, not String object)

// in — checks if property exists (including inherited):
"toString" in {};     // true (inherited from Object.prototype)
0 in [1, 2, 3];      // true (index 0 exists)
"length" in [1, 2];  // true (arrays have .length)

// Comma operator — evaluates all expressions, returns the last:
const x = (1, 2, 3); // x = 3

// void — evaluates expression and returns undefined:
void 0;          // undefined
void "anything"; // undefined

// Ternary (conditional) operator:
const result = condition ? "yes" : "no";

// Grouping operator — controls precedence:
(2 + 3) * 4; // 20 (not 14)
```

## ③ WHY IT EXISTS

JavaScript operators evolved to serve different eras:
- **ES1 (1997):** Basic arithmetic, comparison, logical, bitwise
- **ES6 (2015):** Spread/rest, destructuring — functional programming patterns
- **ES2020:** Optional chaining (`?.`) and nullish coalescing (`??`) — solving the most common null-checking pain points
- **ES2021:** Logical assignment (`&&=`, `||=`, `??=`) — concise conditional assignments

`?.` and `??` were among the most requested features in JavaScript history, solving decades of verbose `if (x && x.y && x.y.z)` patterns.

---

## ④ INTERNAL WORKING

### V8 Bytecode for Operators

```javascript
const x = a ?? b;
```
**V8 Ignition bytecode:**
```
Ldar r0              // Load a
JumpIfNotUndefined [skip]
JumpIfNotNull [skip]
Ldar r1              // Load b (only if a was null/undefined)
[skip]:
Star r2              // Store result in x
```

Optional chaining `a?.b` generates similar short-circuiting bytecode with null/undefined checks.

### Operator Overloading

JavaScript does NOT support custom operator overloading (unlike C++ or Python). The `+` operator's dual behavior (addition + concatenation) is built into the spec, not extensible by users. `Symbol.toPrimitive` is the closest you can get to customizing operator behavior.

---

## ⑤ ADVANTAGES

1. **`??` prevents bugs with valid falsy values** — 0, "", false are preserved.
2. **`?.` eliminates verbose null-checking** — cleaner, safer code.
3. **Destructuring** makes working with complex data structures concise and readable.
4. **Spread/rest** enables immutable patterns and flexible function signatures.

---

## ⑥ DISADVANTAGES / TRAPS

1. **`+` overloading:** `"5" + 3 = "53"` but `"5" - 3 = 2`.
2. **`delete` kills V8 optimization:** Objects switch to slow dictionary mode.
3. **`typeof null === "object"`** — must special-case null checks.
4. **Destructuring with deep nesting can be unreadable:**
   ```javascript
   const { a: { b: { c: { d } } } } = obj; // Too deep — use optional chaining instead
   ```
5. **Spread is shallow:** `{...obj}` doesn't deep-copy nested objects.

## ⑦ PRODUCTION USAGE

```javascript
// Configuration with defaults (??):
const port = process.env.PORT ?? 3000;
const debug = config.debug ?? false;

// Safe property access (?.) + default (??):
const userName = response?.data?.user?.name ?? "Guest";

// Immutable updates (spread):
const updatedUser = { ...user, name: "New Name", updatedAt: Date.now() };
const newItems = [...items, newItem];

// Function with options (destructuring + defaults):
function createServer({
    port = 3000,
    host = 'localhost',
    ssl = false,
    maxConnections = 100,
} = {}) {
    // All options have defaults, parameter itself defaults to {}
}
createServer({ port: 8080 }); // Only override what you need

// Logical assignment for lazy initialization:
class Cache {
    get(key) {
        this.data ??= new Map(); // Initialize on first use
        return this.data.get(key);
    }
}
```

## ⑧ INTERVIEW QUESTIONS

### 🟢 Basic (5)

**Q1: What's the difference between `||` and `??`?** **A:** `||` returns the right side for any falsy value (0, "", false, null, undefined). `??` returns the right side ONLY for `null` or `undefined`. Use `??` when 0 or "" are valid values.

**Q2: What does `?.` do?** **A:** Optional chaining. It short-circuits to `undefined` if the left side is `null` or `undefined`, preventing TypeError for accessing properties of null/undefined.

**Q3: What does the spread operator do?** **A:** `...` expands an iterable into individual elements (in arrays, function args) or copies enumerable own properties (in objects).

**Q4: What's the difference between rest and spread?** **A:** They use the same `...` syntax. Spread expands values (`[...arr]`), rest collects them (`function(...args)`). Spread is in expression position; rest is in parameter/destructuring position.

**Q5: How do you swap two variables without a temp?** **A:** `[a, b] = [b, a]` — array destructuring assignment.

### 🟡 Intermediate (5)

**Q6: What's the output of `"" ?? "default"`?** **A:** `""`. Empty string is NOT null/undefined, so `??` returns the left side. Compare with `"" || "default"` which returns `"default"`.

**Q7: What does `delete` actually do in V8?** **A:** It removes a property from an object and returns `true`. But it often forces V8 to transition the object from fast (hidden class) mode to slow (dictionary) mode, significantly degrading property access performance.

**Q8: What is `typeof` useful for vs. what are its limitations?** **A:** `typeof` is useful for checking primitives and functions. Limitations: `typeof null` returns `"object"`, `typeof []` returns `"object"` (use `Array.isArray`), and it can't distinguish between different object types.

**Q9: How does short-circuit evaluation work with `&&` and `||`?** **A:** `&&` stops at the first falsy value (returns it), or returns the last value if all are truthy. `||` stops at the first truthy value (returns it), or returns the last value if all are falsy. In both cases, subsequent expressions are NOT evaluated.

**Q10: What is destructuring with default values?** **A:** You can set defaults in destructuring: `const { x = 10, y = 20 } = { x: 5 }` gives `x = 5, y = 20`. The default is used when the value is `undefined` (not `null`).

### 🔴 Advanced (5)

**Q11: How does V8 optimize `a?.b?.c` internally?** **A:** V8 generates bytecode with null/undefined checks that short-circuit. TurboFan can optimize repeated optional chaining on the same object shape by caching the hidden class check — if the shape hasn't changed, subsequent `?.` checks can be eliminated.

**Q12: What's the difference between `??=` and `||=`?** **A:** `x ??= 5` assigns 5 only if x is `null` or `undefined`. `x ||= 5` assigns 5 if x is any falsy value (0, "", false, null, undefined). `??=` is safer for preserving valid falsy values.

**Q13: Why is `instanceof` unreliable across frames/realms?** **A:** `instanceof` checks the prototype chain. Different iframes/realms have different built-in constructors. An array from iframe A is NOT `instanceof Array` in iframe B, because `iframe_A.Array !== iframe_B.Array`.

**Q14: How does the `in` operator work internally?** **A:** `"prop" in obj` calls the internal `[[HasProperty]]` operation, which checks the object AND its prototype chain. It returns `true` even for inherited properties. Use `obj.hasOwnProperty("prop")` or `Object.hasOwn(obj, "prop")` for own properties only.

**Q15: Can you use optional chaining for assignment?** **A:** No. `obj?.prop = 5` is a SyntaxError. Optional chaining is read-only. You'd need: `if (obj) obj.prop = 5;` or `obj && (obj.prop = 5);`.

### 🧠 "Explain What Happens Internally" (2)

**Q16: What happens step-by-step when V8 executes `a?.b?.c ?? "default"`?** **A:** 1. Evaluate `a` — if null/undefined, result of `a?.b?.c` is `undefined`. 2. If `a` is not null/undefined, access `a.b` — if null/undefined, result is `undefined`. 3. If `a.b` is not null/undefined, access `a.b.c`. 4. Now evaluate `??`: if the result from steps 1-3 is null/undefined, return `"default"`. Otherwise return the result. V8 compiles this to bytecode with conditional jumps, short-circuiting at each `?.` check.

**Q17: How does object spread (`{...obj}`) differ from `Object.assign` internally?** **A:** Both copy own enumerable properties. Differences: (1) Spread uses `[[OwnPropertyKeys]]` + `[[GetOwnProperty]]` internally; `Object.assign` uses `[[Get]]` which triggers getters. (2) Spread creates a new object; `Object.assign` mutates the target. (3) Spread invokes the setter on the new object (which is a plain object, so it's a simple set); `Object.assign` invokes setters on the target. (4) In V8, spread on an object literal is optimized via `CloneObject` bytecode — it copies the hidden class and all in-object properties in one operation, which can be faster than `Object.assign`'s property-by-property loop.
