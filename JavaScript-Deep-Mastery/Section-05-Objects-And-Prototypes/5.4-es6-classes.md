# 📌 5.4 — ES6 Classes
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## ① DEFINITION

**Plain English:** An ES6 class is a modern, clean blueprint syntax for creating objects with shared behaviors. While it looks like classes in languages like Java or C++, it is not a new OOP model. Under the hood, it is simply a cleaner way to write prototype-based inheritance and constructor functions (often called "syntax sugar").

**Technical Definition:** Introduced in ES6, the `class` keyword defines a constructible function characterized by the internal `[[IsClassConstructor]]: true` flag. Unlike ordinary functions, class constructors cannot be invoked without the `new` operator. Classes encapsulate constructors, prototype methods, static methods, public/private fields (`#field`), and static initialization blocks into a single declarative structure.

**One-line mental model:**
> *ES6 classes are a clean declarative wrapper over JavaScript's prototype delegation system, with strict call checks and native privacy.*

---

## ② DRILL DOWN

### 🔹 A. Class Syntax vs. Prototype Equivalent

To truly understand classes, you must see exactly what prototype code they compile into.

#### 1. The Class Syntax:
```javascript
class User {
    constructor(name) {
        this.name = name;
    }

    greet() {
        return `Hello, ${this.name}`;
    }

    static getSpecies() {
        return "Homo sapiens";
    }
}
```

#### 2. The Prototype Equivalent:
```javascript
function User(name) {
    // 1. Enforce 'new' calling check
    if (!(this instanceof User)) {
        throw new TypeError("Class constructor User cannot be invoked without 'new'");
    }
    this.name = name;
}

// 2. Attach prototype method (non-enumerable!)
Object.defineProperty(User.prototype, 'greet', {
    value: function() { return `Hello, ${this.name}`; },
    writable: true,
    enumerable: false, // Class methods are NOT enumerable!
    configurable: true
});

// 3. Attach static method (directly on the constructor function)
Object.defineProperty(User, 'getSpecies', {
    value: function() { return "Homo sapiens"; },
    writable: true,
    enumerable: false,
    configurable: true
});
```

### 🔹 B. Class Fields, Private Fields, and Methods

Modern JavaScript classes support declarative instance variables and private identifiers using the `#` prefix.

```javascript
class BankAccount {
    // 1. Public Class Field (Initialized per instance)
    branchCode = "NY-101";

    // 2. Private Class Field (Accessible ONLY inside this class)
    #balance = 0;

    constructor(owner, initialDeposit) {
        this.owner = owner;
        this.#balance = initialDeposit; // Allowed inside the class
    }

    // 3. Private Method
    #logTransaction(amount, type) {
        console.log(`[TX]: ${type} of $${amount} processed.`);
    }

    deposit(amount) {
        this.#balance += amount;
        this.#logTransaction(amount, "Deposit");
    }

    getBalance() {
        return this.#balance;
    }
}

const account = new BankAccount("Alice", 1000);
account.deposit(500);

console.log(account.branchCode); // "NY-101"
// console.log(account.#balance); // SyntaxError: Private field '#balance' must be declared in an enclosing class
// account.#logTransaction(500, "Deposit"); // SyntaxError
```

---

### 🔹 C. Class Hoisting and the Temporal Dead Zone (TDZ)

Unlike standard function declarations, classes are **not** hoisted in a usable way. They are bound to a Temporal Dead Zone (TDZ) just like variables declared with `let` or `const`.

```javascript
// Attempting to instantiate before definition:
// const u1 = new User("Alice"); // ReferenceError: Cannot access 'User' before initialization

class User {
    constructor(name) {
        this.name = name;
    }
}
```

---

### 🔹 D. Getters, Setters, and Static Blocks

Classes simplify defining getters/setters and static initialization blocks (useful for complex configuration).

```javascript
class Thermostat {
    #celsius;

    constructor(tempCelsius) {
        this.#celsius = tempCelsius;
    }

    // 1. Getter
    get fahrenheit() {
        return (this.#celsius * 9) / 5 + 32;
    }

    // 2. Setter
    set fahrenheit(value) {
        this.#celsius = ((value - 32) * 5) / 9;
    }

    // 3. Static Initialization Block (Runs ONCE when class is loaded)
    static {
        console.log("Thermostat class initialized in runtime");
        this.defaultUnit = "F";
    }
}

const t = new Thermostat(25);
console.log(t.fahrenheit); // 77
t.fahrenheit = 32;
console.log(t.fahrenheit); // 32
```

## ③ WHY IT EXISTS

1. **Syntactic Standardization**: JavaScript before ES6 suffered from a lack of standardized OOP patterns. Developers mixed constructors, factories, closures, and diverse prototype inheritances, creating highly unmaintainable code. Classes unified these practices into a clean, universally understood standard.
2. **True Encapsulation**: Prior to private fields (`#`), JavaScript had no real way to enforce class privacy. Developers had to use hacks like naming fields with underscores (e.g., `_balance`) or using heavy closures.
3. **Subclassing Built-ins**: ES6 classes changed object allocation to support subclassing built-in constructors (like `Array` or `Error`), which was impossible under legacy prototype structures.

## ④ INTERNAL WORKING: V8 CLASS ENGINE DETAILS

### The `[[IsClassConstructor]]` Internal Flag

If you check the type of a class, it is a function:
```javascript
class Foo {}
console.log(typeof Foo); // "function"
```
However, calling it directly throws:
```javascript
Foo(); // TypeError: Class constructor Foo cannot be invoked without 'new'
```
Internally, V8 sets a boolean flag on the function's internal object: `[[IsClassConstructor]]: true`.
When evaluating a call expression, the interpreter inspects this flag. If the flag is `true` and the operator is not a construct call (`new`), V8 immediately halts execution and throws a `TypeError`.

---

### Private Fields Implementation (Private Symbols / WeakMaps)

JavaScript private fields (`#field`) are not just normal property lookups. They are entirely separated from public properties to prevent dynamic lookups (e.g., `obj["#field"]` is a syntax error).

Internally, V8 implements private fields using **Private Symbols** (which behave like `WeakMap` mappings under the hood):

```
┌────────────────────────────────────────────────────────┐
│                   JSObject Instance                    │
├────────────────────────────────────────────────────────┤
│ • Map (Hidden Class Map pointer)                       │
│ • [Private Symbol(#balance)] ──────────────────────────┼──► Stores value (e.g., 1000)
│ • publicProperty: "Alice"                              │
└────────────────────────────────────────────────────────┘
```

1. When a class is constructed, V8 allocates a unique **Private Symbol** for each private field in the class.
2. The private property is written directly onto the object instance, but the key is the completely hidden, un-inspectable Private Symbol.
3. Because Private Symbols do not show up in reflection APIs (`Object.keys`, `Reflect.ownKeys`, `getOwnPropertySymbols`), they are 100% secure from external introspection.

---

## ⑤ ADVANTAGES

1. **Clear Declarative Syntax**: Drastically reduces prototype boilerplate and organizes object structures into single, legible blocks.
2. **Guaranteed Safety**: Native restrictions prevent dangerous, forgotten `new` calls on constructors.
3. **True Security**: Native private fields (`#`) guarantee perfect privacy, blocking global inspection or injection hacks.
4. **Non-enumerable Methods**: Unlike plain constructors, class methods are automatically created as non-enumerable, keeping object property logs clean.

---

## ⑥ DISADVANTAGES / TRAPS

1. **Illusory Classical OOP**: Developers coming from Java or C++ often assume JS classes use classical copy-inheritance, leading to deep bugs when prototype properties are dynamically mutated at runtime.
2. **Hoisting Traps**: Forgetting that classes are in the TDZ leads to execution crashes if instantiation is coded before declaration.
3. **WeakMap Memory Pitfalls**: Heavy usage of complex private fields and private methods generates extensive internal Private Symbol tracking tables, slightly increasing heap sizes.

## ⑦ PRODUCTION USAGE

### 🛠️ Production-Ready Secure State Manager

Use private fields and static blocks to build secure, robust singletons or state stores:

```javascript
class ConfigurationStore {
    #apiToken;
    #maxConnections = 5;

    constructor(token) {
        if (!token) throw new Error("A secure API token is required.");
        this.#apiToken = token;
    }

    // Secure Getter (only exposes masked token)
    get maskedToken() {
        return `${this.#apiToken.slice(0, 4)}...${this.#apiToken.slice(-4)}`;
    }

    // Static Singleton initializer
    static createDefaultInstance() {
        return new ConfigurationStore(process.env.SECURE_API_KEY ?? "DEFAULT-API-KEY-0000");
    }
}

const config = ConfigurationStore.createDefaultInstance();
console.log(config.maskedToken); // "DEFA...0000"
```

## ⑧ INTERVIEW QUESTIONS

### 🟢 Basic (5)

**Q1: What is the `class` keyword in JavaScript?**  
**A:** A modern ES6 syntactic sugar layer built on top of JavaScript's prototype delegation system to define templates for objects.

**Q2: What is the type of an ES6 class?**  
**A:** It is a function (`typeof ClassName === "function"`).

**Q3: Can you call an ES6 class constructor without the `new` keyword?**  
**A:** No. Attempting to do so immediately throws a `TypeError`.

**Q4: How do you declare a private field in an ES6 class?**  
**A:** Prefix the property name with a hash symbol (e.g., `#balance`).

**Q5: Are methods defined inside an ES6 class enumerable?**  
**A:** No. Class prototype methods are automatically defined with `enumerable: false` to keep object traversals clean.

---

### 🟡 Intermediate (5)

**Q6: Explain the difference between function hoisting and class hoisting.**  
**A:** Function declarations are fully hoisted (both the name and the function body are usable before the declaration line). Classes are hoisted but placed in a **Temporal Dead Zone (TDZ)**. You cannot access or instantiate a class before its actual declaration line in the code, or it throws a `ReferenceError`.

**Q7: How does V8 prevent you from calling a class constructor without the `new` keyword?**  
**A:** V8 sets an internal slot flag `[[IsClassConstructor]]: true` on the compiled class function object. During function invocation resolution, if the engine evaluates a regular call (`[[Call]]`) instead of a construct call (`[[Construct]]`) and sees this flag set to `true`, it halts execution and throws a `TypeError`.

**Q8: Explain the difference between a static method and an instance method in a class.**  
**A:** An instance method is attached to the class prototype (`Class.prototype.method`) and is shared by all instances. A static method is attached directly to the constructor function itself (`Class.method`) and can only be called directly on the class, not on its instances.

**Q9: What are static initialization blocks, and why would you use them?**  
**A:** Static initialization blocks (declared with `static {}`) run exactly once when the class is loaded by the engine. They allow executing complex configuration, initializing static private variables, or binding global event interceptors before any instances are instantiated.

**Q10: Why does `class Foo { x = 5; }` differ from `class Foo { constructor() { this.x = 5; } }` internally?**  
**A:** Conceptually they are very similar, but class fields are evaluated and assigned inside the constructor *before* the constructor body executes. If the constructor has parameter bindings or complex setups, class fields are guaranteed to initialize first.

---

### 🔴 Advanced (5)

**Q11: Explain how private fields (`#field`) are implemented in V8. How does this differ from the standard public property lookup mechanism?**  
**A:** Standard public properties are looked up dynamically via string/symbol keys inside the object Map (hidden class). Private fields use a completely different system. V8 creates unique internal **Private Symbols** unique to the class. When an instance is initialized, the private values are stored on the object using these un-inspectable Private Symbols as keys. Since reflection methods like `Reflect.ownKeys()` or `Object.getOwnPropertySymbols()` explicitly exclude Private Symbols, the private fields remain completely encapsulated and inaccessible outside class scope.

**Q12: Walk through the compilation and execution steps V8 takes when a class declaration is parsed.**  
**A:** 
1. **Parsing**: The engine parses the class syntax into AST nodes representing class fields, methods, and constructor logic.
2. **SFI Creation**: It creates a `SharedFunctionInfo` (SFI) for the class function and flags `[[IsClassConstructor]]: true`.
3. **Prototype Configuration**: It creates the class `.prototype` object, maps class methods to it, and defines them as `writable: true, enumerable: false, configurable: true`.
4. **TDZ Binding**: It binds the class identifier in the active Lexical Environment, setting it as uninitialized (entering the Temporal Dead Zone).
5. **Evaluation**: When execution reaches the class declaration line, the TDZ restriction is removed, and the class variable is bound to the constructor function heap object.

**Q13: How can you access private fields of an instance from *outside* the instance, but *inside* the class? Write an example and explain the security boundary.**  
**A:** JavaScript private fields enforce **class-private** encapsulation rather than **instance-private** encapsulation. This means instances of the same class can access each other's private fields:
```javascript
class Peer {
    #secret;
    constructor(secret) { this.#secret = secret; }
    inspectPeer(peerInstance) {
        return peerInstance.#secret; // Completely valid!
    }
}
const p1 = new Peer("A");
const p2 = new Peer("B");
p1.inspectPeer(p2); // Returns "B"
```
*Security Boundary:* Access is restricted solely by the physical boundary of the class declaration file/module. The parser evaluates the `#` checks lexically during AST compiling; as long as the token is inside the `class` wrapper, it is valid.

**Q14: Explain why class fields defined as arrow functions (`myMethod = () => {}`) are considered a performance and memory anti-pattern.**  
**A:** When you write `myMethod = () => {}` as a class field, it is **not** defined on `Class.prototype`. Instead, it acts as an instance field initialized inside the constructor. Every time you run `new Class()`, a brand new, unique arrow function object is allocated on the heap. This completely bypasses the prototype system, bloating memory and increasing GC load just like old constructor-scoped functions.

**Q15: Explain how a getter and a setter inside a class are represented in the object's descriptor map compared to a regular property.**  
**A:** A regular property is a **Data Descriptor** containing a single slot for `value`. Getters and setters are compiled into an **Accessor Descriptor** on the class prototype object. Instead of holding a value, the prototype descriptor maps the property name to `get` and `set` pointers, which point to the compiled `JSFunction` objects for the getter and setter. Any write or read to the property automatically triggers these functions via the engine's internal `[[Get]]` and `[[Set]]` methods.

### 🧠 "Explain What Happens Internally" (2)

**Q16: Explain the exact memory layouts and hidden class transitions that occur when you instantiate a class containing three public class fields (e.g., `class A { x = 1; y = 2; z = 3; }`).**  
**A:** 
1. **Allocation**: `new A()` allocates a plain heap object. V8 checks `expected_nof_properties` (which is 3) and reserves 3 in-object slots in the instance header.
2. **Hidden Class `C0`**: Initially, the object points to the base Map for class `A` (empty).
3. **Transition to `C1`**: The class field initializer evaluates `x = 1`. V8 updates the instance's map pointer to Map `C1` (stores `x` at offset 0).
4. **Transition to `C2`**: The field initializer evaluates `y = 2`. The map changes to Map `C2` (stores `y` at offset 1).
5. **Transition to `C3`**: The field initializer evaluates `z = 3`. The map changes to Map `C3` (stores `z` at offset 2).
6. **Constructor Exec**: Only after these fields are assigned and the final Map `C3` is reached does V8 execute the custom constructor body. If the constructor assigns no further properties, all instances of `A` share the highly optimized, monomorphic Map `C3`.

**Q17: Explain the exact mechanics of how V8 prevents prototype pollution when using private class fields, compared to using standard string-keyed properties.**  
**A:** Prototype pollution operates by injecting malicious property paths (like `__proto__.isAdmin = true`) into standard objects.
1. When resolving a standard lookup `obj[key]`, V8 uses string matching to crawl the prototype chain. If `__proto__` is resolved, it follows the pointer to the prototype object.
2. When compiling a private field reference `obj.#privateVal`, V8 bypasses the dynamic string-lookup system completely. The compiler hardcodes a check against the unique **Private Symbol** associated with `#privateVal`.
3. V8 requires the target object to possess the private symbol directly on its instance (enforced via internal brand checks).
4. If an attacker pollution payload inserts `#privateVal` into `Object.prototype`, V8's brand check halts execution. Because the prototype object lacks the private brand association, it throws a `TypeError: Cannot read private member from an object whose class did not declare it`. Private fields are entirely immune to prototype pollution.
