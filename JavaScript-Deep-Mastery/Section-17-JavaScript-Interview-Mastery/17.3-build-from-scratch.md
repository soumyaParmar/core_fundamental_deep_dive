### 🔹 A. Implementing a Promises/A+ Library from Scratch

A Promise library must manage three states (PENDING, FULFILLED, REJECTED) and execute callback microtasks asynchronously when settled.

```javascript
const PENDING = "PENDING";
const FULFILLED = "FULFILLED";
const REJECTED = "REJECTED";

class CustomPromise {
    #state = PENDING;
    #value = null;
    #reason = null;
    #onFulfilledCallbacks = [];
    #onRejectedCallbacks = [];

    constructor(executor) {
        const resolve = (value) => {
            if (this.#state === PENDING) {
                this.#state = FULFILLED;
                this.#value = value;
                // Execute callbacks on the microtask queue!
                this.#onFulfilledCallbacks.forEach(cb => queueMicrotask(() => cb(value)));
            }
        };

        const reject = (reason) => {
            if (this.#state === PENDING) {
                this.#state = REJECTED;
                this.#reason = reason;
                this.#onRejectedCallbacks.forEach(cb => queueMicrotask(() => cb(reason)));
            }
        };

        try {
            executor(resolve, reject);
        } catch (error) {
            reject(error);
        }
    }

    then(onFulfilled, onRejected) {
        // Enforce default callbacks if not provided
        onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : val => val;
        onRejected = typeof onRejected === 'function' ? onRejected : err => { throw err; };

        return new CustomPromise((resolve, reject) => {
            const handleFulfilled = () => {
                try {
                    const x = onFulfilled(this.#value);
                    x instanceof CustomPromise ? x.then(resolve, reject) : resolve(x);
                } catch (e) {
                    reject(e);
                }
            };

            const handleRejected = () => {
                try {
                    const x = onRejected(this.#reason);
                    x instanceof CustomPromise ? x.then(resolve, reject) : resolve(x);
                } catch (e) {
                    reject(e);
                }
            };

            if (this.#state === FULFILLED) {
                queueMicrotask(handleFulfilled);
            } else if (this.#state === REJECTED) {
                queueMicrotask(handleRejected);
            } else {
                this.#onFulfilledCallbacks.push(handleFulfilled);
                this.#onRejectedCallbacks.push(handleRejected);
            }
        });
    }
}
```

### 🔹 B. Implementing a Custom EventEmitter

```javascript
class CustomEventEmitter {
    constructor() {
        this.events = new Map();
    }

    on(event, listener) {
        if (typeof listener !== 'function') throw new TypeError("Listener must be a function.");
        if (!this.events.has(event)) this.events.set(event, []);
        this.events.get(event).push(listener);
        return () => this.off(event, listener); // Unsubscribe helper
    }

    emit(event, ...args) {
        const listeners = this.events.get(event);
        if (!listeners) return false;
        // Slice copy to prevent array mutations *during* iteration!
        listeners.slice().forEach(listener => listener(...args));
        return true;
    }

    off(event, listener) {
        const listeners = this.events.get(event);
        if (!listeners) return;
        const index = listeners.indexOf(listener);
        if (index !== -1) listeners.splice(index, 1);
    }
}
```

```javascript
Array.prototype.customReduce = function(callback, initialValue) {
    if (this == null) throw new TypeError("Array.prototype.customReduce called on null or undefined");
    if (typeof callback !== 'function') throw new TypeError(callback + " is not a function");

    const O = Object(this);
    const len = O.length >>> 0;
    let accumulator = initialValue;
    let k = 0;

    if (accumulator === undefined) {
        // Find the first initialized index in sparse arrays!
        let kPresent = false;
        while (k < len && !kPresent) {
            kPresent = k in O;
            if (kPresent) accumulator = O[k];
            k++;

        if (!kPresent) throw new TypeError("Reduce of empty array with no initial value");

    for (; k < len; k++) {
        if (k in O) {
            accumulator = callback(accumulator, O[k], k, O);
        }
    }
```

## ④ INTERNAL WORKING: HOW THESE IMPLEMENTATIONS BIND IN THE ENGINE RUNTIME AND INTERACT WITH THE MICROTASK QUEUE

### Microtask Schedulers and Event Loop Bindings in Rebuilt Primitives

When our custom implementations execute:

#### 1. Promise Resolution via `queueMicrotask`
In our `CustomPromise`:
* Standard async callbacks are pushed to `queueMicrotask(() => callback())`.
* **V8 Heap Queue:** `queueMicrotask` bypasses standard browser macro-timers like `setTimeout`. It registers the callback directly on the **Microtask Queue** of the active V8 `Isolate` instance.
* **Stack Execution:** When the current synchronous JS execution stack finishes, the engine pops the callbacks contiguously.
* **Chaining Resolve:** In `.then()` chains, if a callback returns another Promise `x`, the engine stops executing subsequent callbacks, schedules a new Promise Reaction, and yields execution until `x` settles, maintaining absolute spec compatibility.

#### 2. Safe Array Traversal and Elements-Kind in customMap
* **Smi Element Arrays:** When you invoke `customMap` on `[1, 2, 3]`, V8 identifies the elements kind as `PACKED_SMI_ELEMENTS`.
* **Sparse Array Transition:** If our implementation checks `if (i in O)`, V8 checks the array's backing store map. If empty slots exist, checking `i in O` prevents allocating a `HeapNumber` for `undefined`, keeping array processing fast and monomorphic.
* **Array Length Cast (`len >>> 0`):** By executing `O.length >>> 0`, the engine converts any floating-point or non-numeric length boundary safely into a standard 32-bit unsigned integer, matching the physical memory offset arrays of V8's C++ layer.

## ⑥ DISADVANTAGES / TRAPS

1. **Missing V8 Engine Micro-Optimizations**: Standard JS Promises are compiled directly into optimized **C++ stubs** inside the V8 engine, executing in less than 2 nanoseconds. Rebuilding Promises in JavaScript (even if spec-compliant) runs inside interpreter bytecodes, resulting in a **5x to 10x performance slowdown**.
2. **Memory Retention Leaks in EventEmitters**: If your custom emitter retains a large array of callback closures globally, any variables captured by those closures cannot be garbage collected, leading to slow memory leaks.
3. **Prototype Pollution via Arrays**: Rewriting `Array.prototype` globally can pollute the global scope. If third-party packages iterate over arrays using `for...in` loops, they will read your custom prototype methods (e.g. `customMap`) as object keys, throwing runtime errors.
4. **Recursion Stack Bloat**: A poorly written promise resolver (`resolvePromise`) can recursively invoke nested promise resolutions, leading to a `RangeError: Maximum call stack size exceeded` in deep thenable structures.

### 🛠️ Production-Ready Performance-Optimized Custom Promises Engine

This production-grade script builds an asynchronous execution engine conforming to the Promises/A+ spec. It implements robust exception catching, handles nested promises, protects against cyclic resolutions (e.g. `promise === x`), and utilizes `queueMicrotask` for native microtask alignment:

```javascript
'use strict';

const STATE_PENDING = "PENDING";
const STATE_FULFILLED = "FULFILLED";
const STATE_REJECTED = "REJECTED";

class SafeCustomPromise {
    #state = STATE_PENDING;
    #value = undefined;
    #reason = undefined;
    #fulfilledQueue = [];
    #rejectedQueue = [];

    constructor(executor) {
        if (typeof executor !== 'function') {
            throw new TypeError("Promise resolver must be a function.");
        }

        const resolve = (value) => {
            if (this.#state !== STATE_PENDING) return;
            this.#state = STATE_FULFILLED;
            this.#value = value;
            this.#flush(this.#fulfilledQueue, value);
        };

        const reject = (reason) => {
            if (this.#state !== STATE_PENDING) return;
            this.#state = STATE_REJECTED;
            this.#reason = reason;
            this.#flush(this.#rejectedQueue, reason);
        };

        try {
            executor(resolve, reject);
        } catch (error) {
            reject(error);
        }
    }

    #flush(queue, payload) {
        while (queue.length > 0) {
            const callback = queue.shift();
            // Schedule execution on the native V8 microtask queue!
            queueMicrotask(() => callback(payload));
        }
    }

    then(onFulfilled, onRejected) {
        onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : val => val;
        onRejected = typeof onRejected === 'function' ? onRejected : err => { throw err; };

        const nextPromise = new SafeCustomPromise((resolve, reject) => {
            const executeFulfilled = (value) => {
                try {
                    const result = onFulfilled(value);
                    this.#resolvePromise(nextPromise, result, resolve, reject);
                } catch (error) {
                    reject(error);
                }
            };

            const executeRejected = (reason) => {
                try {
                    const result = onRejected(reason);
                    this.#resolvePromise(nextPromise, result, resolve, reject);
                } catch (error) {
                    reject(error);
                }
            };

            if (this.#state === STATE_FULFILLED) {
                queueMicrotask(() => executeFulfilled(this.#value));
            } else if (this.#state === STATE_REJECTED) {
                queueMicrotask(() => executeRejected(this.#reason));
            } else {
                this.#fulfilledQueue.push(executeFulfilled);
                this.#rejectedQueue.push(executeRejected);
            }
        });

        return nextPromise;
    }

    #resolvePromise(promise, x, resolve, reject) {
        // Protect against cyclic references (Promises/A+ spec 2.3.1)
        if (promise === x) {
            return reject(new TypeError("Chaining cycle detected for promise!"));
        }

        // Handle nested SafeCustomPromise instances (Promises/A+ spec 2.3.2)
        if (x instanceof SafeCustomPromise) {
            x.then(
                y => this.#resolvePromise(promise, y, resolve, reject),
                r => reject(r)
            );
            return;
        }

        // Handle generic objects or functions (thenables) (Promises/A+ spec 2.3.3)
        if (x !== null && (typeof x === 'object' || typeof x === 'function')) {
            let called = false;
            try {
                const then = x.then;
                if (typeof then === 'function') {
                    then.call(
                        x,
                        y => {
                            if (called) return;
                            called = true;
                            this.#resolvePromise(promise, y, resolve, reject);
                        },
                        r => {
                            if (called) return;
                            called = true;
                            reject(r);
                        }
                    );
                } else {
                    resolve(x);
                }
            } catch (error) {
                if (called) return;
                reject(error);
            }
        } else {
            resolve(x); // Resolve plain primitives
        }
    }
}
```

## ⑧ INTERVIEW QUESTIONS

### 🟢 Basic (5)

**Q1: What are the three states of a Promises/A+ compliant Promise?**  
**A:** **PENDING** (active initial state), **FULFILLED** (settled successfully), and **REJECTED** (settled with a failure reason).

**Q2: What is the purpose of `queueMicrotask` in custom Promise engines?**  
**A:** `queueMicrotask()` schedules the callback to run asynchronously on V8's native **Microtask Queue**, executing immediately after the current synchronous stack empties and before the browser returns to rendering or macrotasks.

**Q3: How does a basic Event Emitter register event listeners?**  
**A:** By using a Map or Object key-value store where the keys represent the event names and the values represent an array of subscriber callback functions: `Map("eventName" -> [cb1, cb2])`.

**Q4: Explain the difference between `Array.prototype.map()` and `Array.prototype.forEach()`.**  
**A:** 
* `forEach()` iterates over the elements of an array and executes a callback for each, returning `undefined`.
* `map()` iterates over the elements, executes a callback, and collects the returned values of each callback into a **brand-new array** of the exact same length.

**Q5: What is a sparse array in JavaScript?**  
**A:** An array containing empty slots or "holes" (e.g. `const arr = [1, , 3]`). These slots are uninitialized, meaning they do not possess keys in the object's backing store map, though they still increase the array's `.length`.

### 🟡 Intermediate (5)

**Q6: Why is using `arguments` inside customized Array methods discouraged? What should you use instead?**  
**A:** The `arguments` object is a specialized array-like object. It lacks array prototype methods, has high initialization overhead, and invalidates JIT optimization states in V8. Modern JS uses **Rest Parameters (`...args`)** to capture variables cleanly as real, optimized Array instances.

**Q7: Explain the threat of mutating an array's length directly during iteration inside customized Array methods.**  
**A:** If a callback mutates the parent array's length (e.g. deleting items or expanding them), a naive loop using a static cached length will access out-of-bounds index offsets, returning `undefined` or skipping elements, leading to logical bugs. Standard specs require validating the key existence (`i in O`) on every iteration.

**Q8: Explain what a "Thenable" is in the context of the Promises/A+ specification.**  
**A:** A thenable is an object or function that defines a **`then` method** (e.g. `{ then: (resolve) => resolve(42) }`). The Promises/A+ spec requires promise resolution procedures to intercept and recursively resolve these objects to ensure compatibility with other Promise libraries (like Bluebird or jQuery).

**Q9: In our custom EventEmitter implementation, why did we slice the listeners array (`listeners.slice()`) during `emit()` execution?**  
**A:** If a registered listener callback invokes the `off(event, listener)` method *during* the execution of the loop, it will mutate the array size. Iterating directly over the mutated array causes the loop to skip the next listener in line. Slicing creates a temporary shallow copy of the active listeners, guaranteeing every listener runs safely.

**Q10: Why does a custom implementation of `Array.prototype.reduce()` throw a `TypeError` if executed on an empty array without an initial value?**  
**A:** According to the ECMAScript specification, if the array is empty and no `initialValue` is provided, the function lacks a starting operand. The accumulator cannot be initialized, leaving it without a value to aggregate, forcing the engine to abort execution.

### 🔴 Advanced (5)

**Q11: Detail what V8 does to JIT Elements Kind optimizations when a custom `customMap` method processes a sparse array containing empty slots. How does checking `i in O` affect performance?**  
**A:** 
1. **Elements Kind Selection:** When you instantiate `[1, , 3]`, V8 classifies it as a **sparse array** (`HOLEY_SMI_ELEMENTS`).
2. **Checking `i in O`:** In `customMap`, checking `i in O` forces V8 to execute a dictionary lookup for key `i` in the object's property backing store.
3. **Optimizations Disabled:** Because sparse arrays use a hash table backing store rather than contiguous memory offsets, JIT compilers cannot use fast register operations.
4. **Dictionary Lookup Cost:** The JIT inline cache is demoted, forcing V8 to perform slow C++ runtime dictionary lookups for every index, reducing traversal speed by up to 20x compared to packed arrays.

**Q13: Explain how the native V8 `Promise.resolve(val)` method differs from `new Promise(res => res(val))` regarding heap allocation and event loop ticks.**  
**A:** 
* **`new Promise(res => res(val))`:** Always allocates a new JS Promise object and a new closure wrapper for `res` and `rej` on the heap, executing the executor callback synchronously in the active frame.
* **`Promise.resolve(val)`:** Is highly optimized. If `val` is already a native V8 Promise, it bypasses new allocations completely and returns the exact same Promise instance directly. This reduces heap allocation churn and saves up to 2 event loop microtask ticks.

**Q14: Explain the differences in V8 compilation, register allocation, and JIT deoptimizations between writing a manual `for` loop traversal vs using a custom `Array.prototype.customReduce` implementation inside hot performance loops.**  
**A:** 
* **Manual `for` Loop:** Highly optimized by TurboFan. It uses a single mutable stack register to track the index and directly reads contiguous memory offsets of the array. The JIT compiler generates inline CPU instructions with zero loop boundaries.
* **`customReduce`:** High overhead. It requires allocating a new execution context for the callback closure on every iteration. Variables like `accumulator` must be copied across stack frames, increasing context switches. JIT compilers struggle to inline the dynamic callbacks, leading to JIT deoptimizations and bailing out to interpreter bytecode under heavy operations.

### 🧠 "Explain What Happens Internally" (2)

**Q16: Explain the exact step-by-step sequence of V8 heap allocations, context transformations, and Event Loop microtask queue ticks that occur when our custom Promise library resolves a `CustomPromise.resolve(42)` inside an active execution stack.**  
**A:** 
1. **Instantiation:** `CustomPromise.resolve(42)` allocates a `CustomPromise` object in the Young Generation heap. Its internal `#state` is written as `FULFILLED` and `#value` is written as `42`.
2. **Method Encounter:** The engine reaches `.then(val => val * 2)`.
3. **Callback Allocation:** A callback function `val => val * 2` is allocated, and its internal scope is linked to the enclosing context.
4. **Queue Push:** Because the state is `FULFILLED`, V8 does not queue the callback on the Promise's internal array. It invokes `queueMicrotask`.
5. **Microtask Queue Entry:** V8's C++ layer allocates a `Microtask` descriptor containing the callback closure pointer and appends it to the isolate's internal Microtask Queue.
6. **Execution Exit:** The synchronous main script exits. The call stack depth drops to 0.
7. **Tick Execution:** V8 enters the microtask loop phase. It pops the `Microtask` descriptor, constructs a fresh execution frame, loads the register `val` with `42`, executes the multiplication, and returns `84` to the caller, finalizing the first async tick.

**Q17: Trace the step-by-step C++ V8 engine states and memory layout allocations when a custom array map method is called inside a hot loop on a highly holey, sparse array, detailing the dictionary transition of elements backing stores.**  
**A:** 
1. **Initial Array Layout:** The compiler processes a sparse array `[1, , 3]`. V8's header defines the Elements Kind as `HOLEY_SMI_ELEMENTS`.
2. **Saturating the Threshold:** The array's empty space size grows. Once holes exceed the density threshold (typically when length > 1,024 and less than 50% slots are filled), V8 transitions the array elements format.
3. **Dictionary Promotion:** V8 discards the contiguous memory pointer and converts the array's elements backing store into a **Dictionary Layout (a C++ hash table)**.
4. **Custom Method Call:** The JIT compiler reaches `Array.prototype.customMap`.
5. **Lookup Loop:** The loop executes `i in O`. V8 cannot read a direct memory address index. It invokes the C++ helper `Dictionary::FindEntry`.
6. **Hash Computation:** V8 computes the hash of the integer index `i`, queries the hash table, and checks for collisions.
7. **Callback Dispatch:** If the entry exists, V8 loads the value and dispatches the callback closure, causing severe CPU cycle bloat and high memory bus bandwidth utilization compared to flat index array lookups.
